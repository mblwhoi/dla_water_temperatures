Sicherheit

Damit die Tabellen Files von der text-db-api gelesen und geschrieben werden können ist es meist nötig, dass man den Tabellen Files und der Verzeichnissen in denen sie liegen mit chmod die entsprechenden rechte gibt.
Das führt jedoch zu dem Problem, dass wenn jemand den Ort eines Tabellen Files weis, er in der Lage dazu ist dieses direkt im Browser einzugeben und somit anzusehen. Das kann bei vertraulichen Daten natürlich sehr störend sein.
Um das zu verhinden, kann man im Verzeichniss wo die Tabellen Files liegen einfach ein .htaccess File erstellen mit folgendem Inhalt:

<Files>
Order deny,allow
Deny from all
</Files>

Und schon ist es nicht mehr möglich die Tabellen Files direkt über den Browser anzusehen.
