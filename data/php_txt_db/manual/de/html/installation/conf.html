Konfiguration

In der Datei const.php können einige Einstellungen vorgenommen werden. Dies ist jedoch nicht zwingend nötig
da man auch die Standardwerte verwenden kann. Man findet die Einstellungen in der Form von Variabeln
in der Datei const.php nach dem Kommentar "User Settings".

Einstellung Standardwert Beschreibung
DEBUG
0
 Wenn man DEBUG auf 1 setzt, so gibt die API zusätzliche Debug Informationen aus. Wenn Ihr ein Problem habt oder einen Bug-Report machen wollt können diese Informationen z.T. hilfreich sein.
LIKE_CASE_SENSITIVE
0
 Wenn man diese Einstellung auf 1 setzt, so unterscheidet LIKE GROSS- und kleinschreibung. Wenn auf 0 gesetzt wird dazwischen nicht unterschieden.
ORDER_CASE_SENSITIVE
0
 Wenn auf 1 gesetzt, wird beim sortieren von Datensätzen mit ORDER BY die GROSS-kleinschreibung unterschieden. D.h. GROSSBUCHSTABEN kommen vor kleinbuchstaben (P -> X -> a -> b).
Wenn auf 0 gesetzt, wir GROSS-kleinschreibung beim sortieren NICHT unterschieden (a -> b -> P -> X)
ASSUMED_RECORD_SIZE
30
 Setzt diesen Wert auf die Ungefähre Durschnittsgrösse eines Datensatzes. Dieser Wert wird intern verwendet, und wenn er weise gewählt wurde kann das positive auswirkungen auf die Performance haben. NICHT auf 0 setzten! Wenn du dir nicht sicher bist lass einfach den Standardwert!!

Diese Einstellung ist NICHT als Begrenzung für die Datensatzgrösse gedacht. Eine solche Begrenzung gibt es nicht!
PRINT_ERRORS
1
 Wenn auf 1 gesetzt werden Fehler angezeigt, andernfalls werden sie unterdrückt. Sofern man keine dynamischen SQL-Statements verwendet, weisen Fehler immer auf ein Fehler des Entwicklers hin, also besser auf 1 lassen! Ausser man erzeugt seine SQL Statement dynamisch...
PRINT_WARNINGS
0
 Wenn auf 1 gesetzt werden Warnungen angezeigt, andernfalls werden sie unterdrückt.
